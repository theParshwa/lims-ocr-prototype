You are a senior software engineer and AI solution architect. Build production-quality, modular, well-documented code.

### OBJECTIVE
Create an AI-powered document processing agent that reads LIMS (Laboratory Information Management System) configuration documents (STP, PTP, and similar document types) from PDF or Word files, intelligently extracts relevant information, and populates a structured Excel Load Sheet with predefined sheets and columns.

The agent must understand LIMS terminology and logically determine which data belongs in which sheet and column.

---

### INPUT
The system must accept:
- PDF files
- Microsoft Word (.docx) files
- Documents may be STP, PTP, or other LIMS configuration documents
- Documents may vary in format and layout

---

### OUTPUT
Generate or update an Excel file with multiple sheets.

Not all sheets will be populated for all document types. The AI must determine applicability.

### CORE FUNCTIONAL REQUIREMENTS

1. Document Parsing
- Extract structured and semi-structured data from PDFs and Word documents.
- Support tables, paragraphs, lists, and headings.
- Use OCR if PDF is scanned.

2. Intelligent Section Mapping
- The AI must understand document context.
- It must decide which sheet each extracted item belongs to.
- Some sheets may not be applicable depending on document type.

3. LIMS Domain Logic
- STP documents typically include test methods, analyses, components, and specifications.
- PTP documents may include product/test planning data.
- Maintain logical relationships between entities (Example: Components must link to Analysis).

4. Data Normalization
- Standardize naming conventions.
- Trim whitespace.
- Normalize units and numerical formats.
- Remove duplicates.

5. Validation Rules
- Ensure required fields are populated.
- Validate numeric fields.
- Ensure references across sheets remain consistent.

6. Confidence Scoring
- Provide a confidence score for extracted data.
- Flag uncertain mappings.

7. Logging
- Maintain processing logs.
- Capture extraction and mapping decisions.

8. Error Handling
- Continue processing even if some sections fail.
- Provide error report output.
