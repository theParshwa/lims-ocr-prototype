04 · AI Extraction Strategy
🤖Combined Extraction Prompt

A single system + user prompt per chunk asks the LLM to return ALL 12 entity types simultaneously in one JSON object. No sequential chaining required.


Before (old): 8 separate LLM calls per chunk
After (current): 1 combined call per chunk


A 20-page document ≈ 2 chunks ≈ 2 API calls total

📐Chunking Parameters

Text is split using LangChain's RecursiveCharacterTextSplitter:

chunk_size12 000 chars (~5 pages)
chunk_overlap200 chars
separators\n\n → \n → space
temperature0.0 (deterministic)
max_tokens8 192
🎓Training / Few-Shot Context

Users upload completed Load Sheet .xlsx files via the Training page. These are parsed (up to 5 rows per sheet), stored in SQLite, and automatically prepended to the first chunk of every new extraction job as few-shot examples.


More examples = higher accuracy for your specific products and analyses.

⚠️Confidence Scoring

Every extracted record carries a confidence float (0–1) returned by the LLM.


≥ 0.85High confidence — no flag
0.6 – 0.84Medium — yellow cell in Excel
< 0.6Low — flagged for review
Validation errorRed cell in Excel

08 · Key Design Decisions
⚡No Celery / Redis

FastAPI's built-in BackgroundTasks handles async processing for single-instance deployments. Celery can be swapped in for scaling with no API changes.

🔁1 LLM Call per Chunk

All 12 entity types extracted in a single combined prompt per chunk. 80× fewer API calls vs the original per-entity approach. Results in ~5× faster processing.

🔌OpenAI GPT-4o

Set OPENAI_API_KEY in .env. The llm_factory.py module handles initialisation.

📦SQLite for Simplicity

Lightweight, zero-configuration, file-based DB via aiosqlite. Full result JSON blobs stored per job. Can be replaced with PostgreSQL for multi-user production deployments.

🎓RAG-style Training

Completed Load Sheets are parsed and stored as few-shot context. On each new job, examples are injected into the first chunk prompt — improving accuracy without fine-tuning.

🖊️Inline Editing

AG Grid allows users to edit any cell directly in the browser. Changes trigger PUT /api/jobs/{id}/data which re-runs validation and updates the stored result before export.
